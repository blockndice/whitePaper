02 — Modèle P2E

Simple.
Direct.
Entre joueurs.

La mécanique challenger / attaquant fonctionne en Web2 pur et se branche sur Solana si le projet franchit le cap du succès.

« Web2 d'abord. Web3 si les joueurs sont là. »
OPTION A
pump.fun

Token communautaire sur Solana. Bonding curve automatique, liquidité immédiate, récompenses en token.

OPTION B
x402

Micropaiements USDC machine-à-machine. Escrow auto, zéro token custom, distribution directe.

// décision selon le profil du jeu et la taille de sa communauté

mécanique des défis web2 → web3 si succès
Challenger
Pose un défi

Choisit un objectif précis. Mise une récompense. Attend que quelqu'un relève.

vs
Attaquant
Tente de gagner

N'importe quel joueur peut tenter de remplir le défi dans le temps imparti.

✓ Objectif atteint

L'Attaquant reçoit la récompense. Vérifié automatiquement.

✗ Non atteint

Le Challenger récupère sa mise intégralement. Défi invaincu.

03 — Stack

La bonne tech,
au bon moment.

Un moteur unique pour toutes les plateformes. Web2 en fondation — Web3 conditionné au succès.

MOTEUR UNIQUE — Phase I & II
Web + Mobile · Phase I & II
Solar2D
Moteur Lua open-source. Export HTML5 pour itch.io et navigateur, build natif iOS/Android pour mobile. Une seule codebase, toutes les plateformes. Physique Box2D, audio, sprites intégrés.
Distribution · Phase I
itch.io
Publication HTML5 via export Solar2D. Audience indie, analytics, zéro friction.
BACKEND — Phase II
Backend · Phase II
Supabase
Leaderboards temps réel, profils joueurs, métriques d'engagement. Partagé entre version web et mobile Solar2D.
WEB3 — Phase III · conditionnel au succès Web2
P2E · Option A
pump.fun
Token communautaire sur Solana. Bonding curve automatique, liquidité immédiate. Récompenses en token natif.
P2E · Option B
x402
Protocole HTTP micropaiements USDC. Escrow automatique challenger/attaquant. Zéro token custom requis.
Infra · Phase III
Solana · Anchor
Smart contracts Rust audités. Wallet Phantom. Frais quasi-nuls, exécution instantanée.
